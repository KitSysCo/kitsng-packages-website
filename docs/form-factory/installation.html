Installation & Setup

This guide will help you install and configure kitsng-form-factory in your Angular project.

Prerequisites

  • Angular 20.3.0+ (or compatible version)
  • PrimeNG 20.3.0+ (for UI components)
  • RxJS
  • @angular/forms
  • google-libphonenumber (for input-phone field)
  • @angular/google-maps (for google-map field)
  • quill (for editor field)

Installation

Step 1: Install the Package

npm install kitsng-form-factory

Or if using a local package:

npm install ./path/to/kitsng-form-factory

Step 2: Import the Module

In your Angular module (e.g., app.module.ts):

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { ReactiveFormsModule } from '@angular/forms';


// PrimeNG Modules (required for form fields)
import { InputTextModule } from 'primeng/inputtext';
import { DropdownModule } from 'primeng/dropdown';
import { CalendarModule } from 'primeng/calendar';
// ... import other PrimeNG modules as needed



// kitsng-form-factory
import { KitsngFormFactoryModule } from 'kitsng-form-factory';



@NgModule({
  imports: [
    BrowserModule,
    ReactiveFormsModule,
    // PrimeNG modules
    InputTextModule,
    DropdownModule,
    CalendarModule,
    // ... other PrimeNG modules
    KitsngFormFactoryModule
  ],
  // ...
})
export class AppModule { }

Step 3: Tailwind CSS Support

The library supports Tailwind CSS utility classes. Make sure you have Tailwind CSS installed and configured in your Angular project.

If you're using Tailwind CSS, the library components will work seamlessly with Tailwind utility classes.

Step 4: Basic Usage

In your component:

import { Component } from '@angular/core';
import { FormGroup } from '@angular/forms';
import { KitsngFormFactoryModel } from 'kitsng-form-factory';


@Component({
  selector: 'app-my-form',
  template: 
    
    
  
})
export class MyFormComponent {
  myForm = new FormGroup({});
  
  fields: KitsngFormFactoryModel[] = [
    {
      controlType: 'input',
      colSize: 'col-span-12',
      options: {
        formControlName: 'name',
        label: 'Name',
        validators: { required: true }
      }
    }
  ];
}

PrimeNG Modules Required

Depending on which form fields you use, you'll need to import the corresponding PrimeNG modules:

Field Type PrimeNG Module
input InputTextModule
password PasswordModule
textarea InputTextareaModule
dropdown DropdownModule
calendar-picker CalendarModule
checkbox CheckboxModule
multi-select MultiSelectModule
radio-button RadioButtonModule
select-button SelectButtonModule
input-number InputNumberModule
file-upload FileUploadModule
editor EditorModule
slider SliderModule
switch-button ToggleButtonModule
tree-select TreeSelectModule
auto-complete AutoCompleteModule
color-picker ColorPickerModule
button ButtonModule
divider DividerModule
icon-picker IconFieldModule, InputIconModule
Note: You only need to import the modules for the fields you actually use in your forms.

CSS Dependencies

PrimeNG Styles (Required)

Make sure you have PrimeNG CSS imported in your angular.json or styles.css:

"styles": [
  "node_modules/primeng/resources/themes/your-theme/theme.css",
  "node_modules/primeng/resources/primeng.min.css",
  "node_modules/primeicons/primeicons.css",
  "styles.css"
]

Tailwind CSS (Optional)

If you're using Tailwind CSS, make sure it's properly installed and configured in your Angular project. The library components work seamlessly with Tailwind utility classes.

Additional Configuration

For Zoneless Angular Applications

If you're using zoneless Angular, the library is fully compatible. Just bootstrap your app with:

import { provideExperimentalZonelessChangeDetection } from '@angular/core';


bootstrapApplication(AppComponent, {
  providers: [
    provideExperimentalZonelessChangeDetection(),
    // ... other providers
  ]
});

See Zoneless Compatibility for more details.
