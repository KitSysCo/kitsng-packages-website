Input Phone with Custom Data Provider Example

This example shows how to use the input-phone component with a custom data provider to load countries from an external source.

Approach 1: Using Generic Data Provider (Recommended)

Step 1: Create a Custom Country Data Provider

Approach 4: Using Static Countries from google-libphonenumber

If you don't have a data provider and want to use built-in country codes from google-libphonenumber:

{
  controlType: 'input-phone',
  colSize: 'col-span-12',
  options: {
    formControlName: 'phoneNumber',
    label: 'Phone Number',
    placeholder: 'Enter phone number',
    // Use google-libphonenumber to get countries
    staticCountriesCode: true,
    validators: {
      required: true
    }
  }
}
Note: This will only be used if no provider is available (no dataProvider, no loadData, and no entity/apiBaseUrl).

Approach 5: Using Legacy API (Backward Compatible)

If you're using the existing API structure:

{
  controlType: 'input-phone',
  colSize: 'col-span-12',
  options: {
    formControlName: 'phoneNumber',
    label: 'Phone Number',
    placeholder: 'Enter phone number',
    // Legacy API approach
    entity: 'Countries',
    apiBaseUrl: 'https://api.example.com',
    module: 'SharedData',
    version: 'v1.2',
    query: {
      active: true
    },
    memoryCache: true, // Cache countries in memory
    validators: {
      required: true
    }
  }
}

Expected Country Data Format

The input-phone component expects countries to have the following structure:

interface Country {
  CountryCode: string;  // Required: Phone country code (e.g., "1", "44", "971")
  id: string;          // Required: Country identifier for flag display (e.g., "us", "gb", "ae")
  // Optional: Any other properties you want to include
  name?: string;
  flag?: string;
}

Custom Data Mapping

The mappOptions function allows you to transform your API response to match the expected format:

mappOptions: (countries) =&gt; {
  return countries.map((country: any) =&gt; {
    // Transform your API response to expected format
    return {
      ...country,
      // Map phone code field (adjust based on your API)

                   country.code,
      // Map country ID for flag display

          country.isoCode?.toLowerCase()
    };
  });
}

Priority Order

The component will try to load countries in this order:

  1. Generic Data Provider (dataProvider) - Highest priority
  2. Load Data Function (loadData)
  3. Legacy API (entity/apiBaseUrl)
  4. Static Countries (staticCountriesCode: true) - Only if no provider is available
  5. Fallback to Legacy API with default entity 'Countries'

Notes

  • The component automatically handles loading states and errors
  • Countries are loaded once and cached (if memoryCache: true)
  • The phone number validation uses Google's libphonenumber library
  • The country code dropdown shows flags if you have the flag CSS classes set up
  • You can customize the country data format using mappOptions
  • When using staticCountriesCode: true, countries are loaded from google-libphonenumber's supported regions
  • Static countries include all regions supported by google-libphonenumber (200+ countries)
  • Countries are sorted by country code when using static countries
