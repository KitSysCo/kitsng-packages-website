Input Phone with Static Countries from google-libphonenumber

This example shows how to use the input-phone component with built-in country codes from google-libphonenumber library.

Basic Usage

When you don't have a data provider and want to use the built-in country codes:

import { Component, OnInit } from '@angular/core';
import { FormGroup, FormBuilder } from '@angular/forms';
import { KitsngFormFactoryModel } from 'kitsng-form-factory';


@Component({
  selector: 'app-phone-example',
  template: 
    
    
  
})
export class PhoneExampleComponent implements OnInit {
  form: FormGroup;
  fields: KitsngFormFactoryModel[] = [];



constructor(private fb: FormBuilder) {
    this.form = this.fb.group({});
  }



ngOnInit() {
    this.fields = [
      {
        controlType: 'input-phone',
        colSize: 'col-span-12',
        options: {
          formControlName: 'phoneNumber',
          label: 'Phone Number',
          placeholder: 'Enter phone number',
          // Use google-libphonenumber to get countries
          staticCountriesCode: true,
          validators: {
            required: true
          }
        }
      }
    ];
  }
}

How It Works

  1. No Provider Required: When staticCountriesCode: true is set and no data provider is available, the component automatically uses google-libphonenumber to get all supported country codes.
  1. Automatic Fallback: If you have a provider configured but it fails, and staticCountriesCode: true is set, it will automatically fallback to google-libphonenumber countries.
  1. Complete Coverage: Includes all 200+ countries/regions supported by google-libphonenumber.
  1. Sorted by Code: Countries are automatically sorted by country code for easy selection.

Priority Order

The component checks for data sources in this order:

  1. dataProvider - Custom data provider (highest priority)
  2. loadData - Load data function
  3. entity/apiBaseUrl - Legacy API
  4. staticCountriesCode: true - google-libphonenumber (only if no provider above)
  5. Fallback to legacy API with default entity

Example with Custom Mapping

You can still use mappOptions to customize the country data format:

{
  controlType: 'input-phone',
  colSize: 'col-span-12',
  options: {
    formControlName: 'phoneNumber',
    label: 'Phone Number',
    staticCountriesCode: true,
    // Custom mapping (optional)
    mappOptions: (countries) => {
      return countries.map((country: any) => ({
        ...country,
        // Add custom properties
        displayName: ${country.name} (+${country.CountryCode})
      }));
    },
    validators: {
      required: true
    }
  }
}

Benefits

  • ✅ No API Required: Works without any external data source
  • ✅ Always Available: Countries are always loaded, no network dependency
  • ✅ Up-to-date: Uses google-libphonenumber's maintained country data
  • ✅ Fast: No network requests, instant loading
  • ✅ Complete: Includes all supported countries/regions

Notes

  • The staticCountriesCode option only works when no other provider is configured
  • Country names are mapped from region codes (e.g., 'US' → 'United States')
  • Countries are sorted by country code (1, 7, 20, 27, etc.)
  • The component uses the same validation library (google-libphonenumber) for consistency
