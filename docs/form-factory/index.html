Overview

kitsng-form-factory is a comprehensive Angular library that allows you to build dynamic, reactive forms with minimal code. It provides:
  • ✅ 30+ Form Field Types - From simple inputs to complex lookup components
  • ✅ Advanced Validation - Built-in, custom, and async validators
  • ✅ Conditional Logic - Dynamic visibility, readonly, and disabled states
  • ✅ Field Dependencies - Fields that react to other field changes
  • ✅ Form Groups & Arrays - Nested structures with tabs, steppers, and dynamic arrays
  • ✅ Generic Lookup Support - Works with any API structure (see Generic Lookup Guide)
  • ✅ Tailwind CSS Support - Full Tailwind utility class support
  • ✅ Zoneless Angular Compatible - Works with zoneless Angular applications (see Zoneless Compatibility)
  • ✅ Accessibility - WCAG compliant with ARIA attributes
  • ✅ Type Safety - Full TypeScript support
  • ✅ Extensible - Register custom field types at runtime

Key Features

1. Dynamic Form Generation

Build complex forms from configuration objects - no template code needed.

2. Validation System

  • Built-in validators (required, email, min, max, etc.)
  • Custom synchronous validators
  • Asynchronous validators (server-side validation)
  • Dynamic validation based on form state

3. Conditional Logic

  • Show/hide fields based on other field values
  • Enable/disable fields dynamically
  • Set readonly state conditionally
  • Multiple condition operators (equals, greaterThan, isEmpty, etc.)

3.5. Control Layout Options

  • Horizontal Layout - Label and control side-by-side
  • Vertical Layout - Label above control (default)
  • Custom column sizes for labels and controls
  • Flexible container classes for styling

4. Field Dependencies

  • Fields can depend on other fields
  • Dynamic options based on dependencies
  • Automatic updates when dependencies change

5. Lifecycle Hooks

  • onInit - Called when form/field initializes
  • onDestroy - Called when form/field is destroyed
  • onFieldChange - Called when any field changes
  • ngModelChange - Enhanced change handlers

6. Generic Architecture

  • Works with any API structure
  • Pluggable data providers
  • No hardcoded dependencies
  • Fully customizable

7. Tailwind CSS Support

  • Full Tailwind utility class support
  • Uses Tailwind grid system and utility classes

8. Zoneless Angular Support

  • Compatible with zoneless Angular applications
  • Uses OnPush change detection strategy
  • Explicit change detection triggers
  • Works seamlessly with or without Zone.js

Quick Example

import { Component } from '@angular/core';
import { FormGroup } from '@angular/forms';
import { KitsngFormFactoryModel } from 'kitsng-form-factory';


@Component({
  selector: 'app-my-form',
  template: 
    
    
  
})
export class MyFormComponent {
  myForm = new FormGroup({});



formFields: KitsngFormFactoryModel[] = [
    {
      controlType: 'input',
      colSize: 'col-span-12',
      options: {
        formControlName: 'name',
        label: 'Full Name',
        placeholder: 'Enter your name',
        // Vertical layout (default) - label above control
        controlLayout: 'vertical',
        validators: {
          required: true,
          minLength: 3
        }
      }
    },
    {
      controlType: 'dropdown',
      colSize: 'col-span-12',
      options: {
        formControlName: 'country',
        label: 'Country',
        placeholder: 'Select country',
        // Horizontal layout - label and control side-by-side
        controlLayout: 'horizontal',
        labelColSize: 'col-span-3',  // Label takes 3 columns
        controlColSize: 'col-span-9', // Control takes 9 columns
        dropdownOptions: [
          { id: 1, text: 'USA' },
          { id: 2, text: 'Canada' }
        ],
        validators: {
          required: true
        }
      }
    }
  ];



onSubmit() {
    if (this.myForm.valid) {
      console.log('Form Data:', this.myForm.value);
    }
  }



onFieldChange(event: any) {
    console.log('Field changed:', event);
  }
}

Control Layout

The controlLayout option controls how labels and controls are positioned:

  • vertical (default): Label appears above the control
  • horizontal: Label and control appear side-by-side

When using horizontal layout, you can customize column sizes:

  • labelColSize: Tailwind CSS class for label column (e.g., 'col-span-3')
  • controlColSize: Tailwind CSS class for control column (e.g., 'col-span-9')
  • horizontalContainerClass: Custom CSS class for the horizontal container

Installation

npm install kitsng-form-factory
Peer Dependencies:
  • Angular 20.3.0+
  • PrimeNG 20.3.0+
  • google-libphonenumber (for input-phone field)
  • @angular/google-maps (for google-map field)
  • quill (for editor field)

See Installation Guide for detailed setup instructions.

Key Features in Detail

Validation System

  • Built-in Validators: required, email, min, max, minLength, maxLength, pattern, requiredTrue
  • Custom Validators: Provide your own validation functions
  • Async Validators: Server-side validation support with Promises or Observables
  • Dynamic Validation: Validation rules can change based on form state

Conditional Logic

  • Visibility Control: Show/hide fields with isVisible, isHidden, or visibleWhen
  • Readonly State: Dynamic readonly state with boolean or function
  • Disabled State: Dynamic disabled state with boolean or function
  • Operators: equals, notEquals, includes, greaterThan, lessThan, isEmpty, isNotEmpty, etc.

Field Dependencies

  • Fields can depend on other fields using dependsOn
  • Dynamic options based on dependencies with dynamicOptions
  • Computed values with dynamicValue
  • Automatic updates when dependencies change

Lifecycle Hooks

  • onInit - Called when form/field initializes
  • onDestroy - Called when form/field is destroyed
  • onFieldChange - Called when any field changes
  • ngModelChange - Enhanced change handlers with full context

Generic Lookup Components

The lookup components (lookup-select and lookup-multi-select) support three approaches:
  1. Generic Data Provider (Recommended) - Implement ILookupDataProvider interface
  2. Custom Load Function - Provide a function that returns an Observable
  3. Legacy API Structure - Backward compatible with entity/module/version structure

See Generic Lookup Guide for detailed examples.
