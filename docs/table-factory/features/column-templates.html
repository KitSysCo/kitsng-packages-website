Column Templates

Custom rendering for table columns using templates.

Overview

Column templates allow you to customize how data is displayed in table cells. There are two methods to provide templates:

  1. HTML Directive - Pass templates directly in HTML
  2. TypeScript Config - Pass templates via component configuration

Method 1: HTML Directive (Recommended)

Pass templates directly in the component template using the kitsngColumnTemplate directive.

Basic Usage

<kitsng-table-factory [tableConfig]="tableConfig">
  <ng-template kitsngColumnTemplate="status" let-value>
    <span 
      class="px-2 py-1 rounded text-sm font-medium"
      [class.bg-green-100]="value === 'active'"
      [class.text-green-800]="value === 'active'"
      [class.bg-red-100]="value === 'inactive'"
      [class.text-red-800]="value === 'inactive'">
      {{ value }}
    </span>
  </ng-template>
</kitsng-table-factory>

Template Context

The template provides three context variables:

  • $implicit or let-value - The cell value
  • let-column="column" - The column configuration object
  • let-rowData="rowData" - The entire row data object

Method 2: TypeScript Config

Pass templates via ViewChild references in your component.

Basic Usage

import { Component, ViewChild, TemplateRef } from '@angular/core';

@Component({
  selector: 'app-users',
  template: 

        {{ value }}

Using ngAfterViewInit

If you need to set the template after view initialization:

import { Component, ViewChild, TemplateRef, AfterViewInit } from '@angular/core';

export class UsersComponent implements AfterViewInit {
  @ViewChild('statusTemplate') statusTemplate!: TemplateRef;
  tableConfig!: KitsngTableConfig;

ngAfterViewInit() {
    this.tableConfig = {
      columns: [
        { field: 'status', header: 'Status', template: this.statusTemplate }
      ],
      data: [...]
    };
  }
}

Template Priority

When both methods are used for the same column:

  • Config template takes priority (higher priority)
  • Directive template is used as fallback
// Config template (will be used)
tableConfig.columns = [
  { field: 'status', template: this.statusTemplate }
];

Advanced Usage

Dynamic Templates

@ViewChild('template1') template1!: TemplateRef;
@ViewChild('template2') template2!: TemplateRef;

getTemplate(field: string): TemplateRef {
  if (field === 'status') return this.template1;
  if (field === 'actions') return this.template2;
  return null;
}

tableConfig: KitsngTableConfig = {
  columns: [
    { 
      field: 'status', 
      template: this.getTemplate('status')
    }
  ]
};

Shared Templates

// In a shared service or base component
export class SharedTemplates {
  static statusTemplate(templateRef: TemplateRef): any {
    return templateRef;
  }
}

Best Practices

  1. Use directive method for component-specific templates
  2. Use config method for reusable/shared templates
  3. Keep templates simple - extract complex logic to component methods
  4. Use context variables - rowData for accessing full row data
  5. Handle null/undefined values in templates
