Overview

Understanding the architecture and concepts of Kitsng Table Factory.

Architecture

Kitsng Table Factory consists of three main components:

  1. KitsngTableFactory - Basic table component for static data
  2. KitsngFetchTableFactory - Table component with API data fetching
  3. KitsngTreeTableFactory - Hierarchical tree table component

Core Concepts

Table Configuration

All components use a KitsngTableConfig object to configure the table:

interface KitsngTableConfig {
  columns: KitsngColumnOption[];  // Column definitions
  data: any[];                    // Table data
  // ... other configuration options
}

Column Configuration

Each column is defined with a KitsngColumnOption:

interface KitsngColumnOption {
  field: string;                  // Data field name
  header: string;                 // Column header
  sortable?: boolean;             // Enable sorting
  template?: TemplateRef;    // Custom template
  visible?: boolean;              // Column visibility
  width?: string;                 // Column width
  // ... other options
}

Data Flow

┌─────────────────┐
│   Data Source │
└───────┬───────┘
        │
        ├─── Static Data ───&gt; KitsngTableFactory
        │
        ├─── API Data ───&gt; KitsngFetchTableFactory
        │
        └─── Tree Data ───&gt; KitsngTreeTableFactory

Key Features

1. Generic API System

Works with any API structure:

fetchApiInfo: {
  fetchFunction: (query) =&gt; this.http.post('/api/data', query),
  responseMapper: (response) =&gt; ({
    items: response.data.items,
    totalItems: response.data.total
  })
}

2. Flexible Templates

Two ways to provide templates:

  • HTML Directive:
  • TypeScript Config: column.template = this.templateRef

3. Event Handling

Rich event system:

  • Row clicks (single, double)
  • Cell clicks
  • Selection changes
  • Page changes
  • Sorting events
  • Row expand/collapse

4. Styling

Self-contained CSS framework:

  • Custom kitsng- prefixed classes
  • No PrimeFlex/Bootstrap dependency
  • Consistent styling

Data Structures

Pagination Response

interface KitsngPaginationResponse {
  items: T[];
  totalItems?: number;
  pageInfo?: {
    totalItems: number;
    totalPages?: number;
    pageIndex?: number;
    pageSize?: number;
  };
}

Table Query

interface KitsngTableQuery {
  pageIndex?: number;
  pageSize?: number;
  search?: string;
  filters?: any;

  [key: string]: any;
}

Best Practices

  1. Use FetchTableFactory for API-driven data
  2. Use TableFactory for static/small datasets
  3. Use TreeTableFactory for hierarchical data
  4. Define columns once and reuse
  5. Use templates for complex rendering
  6. Handle events for user interactions
  7. Configure pagination for large datasets
